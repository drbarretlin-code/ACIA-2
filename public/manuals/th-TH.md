# tuc translator คู่มือการใช้งาน

---

## 1. คำอธิบายฟังก์ชันและการใช้งานอินเทอร์เฟซ (UI)

### 1. การเข้าสู่ระบบ

เมื่อเปิดหน้าเว็บครั้งแรก ให้ดำเนินการตามขั้นตอนดังนี้:

1. **ระบุชื่อผู้ใช้**: ใส่ชื่อที่จะให้แสดงผล (เช่น `Barret` หรือตำแหน่งงานของคุณ)
2. **ระบุ Gemini API Key**: ใส่รหัส API ของ Google AI Studio (รูปแบบเริ่มต้นด้วย `AIza...`)
3. **สร้างหรือเข้าร่วมห้องประชุม**:
   - คลิก "สร้างห้องใหม่" เพื่อเริ่มเซสชันการแปล
   - หรือวางลิงก์เชิญจากผู้อื่นลงในช่องระบุ ID เพื่อเข้าร่วม

> [!IMPORTANT]
> **ระบบคัดกรองการเข้าถึงที่เข้มงวด**: เมื่อเข้าร่วมห้องประชุม ระบบจะตรวจสอบว่าคุณมีรหัส API ที่ใช้งานได้หรือไม่ (รหัสส่วนตัวหรือรหัสที่โฮสต์แบ่งปัน) หากไม่มีทั้งสองอย่าง หน้าจอตั้งค่าจะปรากฏขึ้นโดยอัตโนมัติ หากคุณเลือกยกเลิก คุณจะไม่สามารถเข้าห้องหรือดูผลการแปลใดๆ ได้

> [!NOTE]
> API Key จะถูกเก็บไว้ในเบราว์เซอร์ของคุณเท่านั้น จะไม่มีการอัปโหลดไปเก็บที่เซิร์ฟเวอร์ใดๆ

---

### 2. แถบเลือกภาษา (แผงควบคุม)

แถบเมนูตรงกลางหน้าจอ เรียงจากซ้ายไปขวา ดังนี้:

| องค์ประกอบ | คำอธิบาย |
|---|---|
| 🏳️ ธงชาติฝั่งซ้าย | ภาษาที่คุณพูด (ภาษาต้นทาง) |
| 🌐 เมนูเลือกฝั่งซ้าย | เลือก "ภาษาของฉัน" (Local) ไม่สามารถเปลี่ยนได้ขณะกำลังบันทึกเสียง |
| ⇄ ปุ่มลูกศรตรงกลาง | ปุ่มสลับภาษาซ้าย-ขวาอย่างรวดเร็ว ไม่สามารถใช้งานได้ขณะกำลังบันทึกเสียง |
| 🌐 เมนูเลือกฝั่งขวา | เลือก "ภาษาของอีกฝ่าย" (Client) ไม่สามารถเปลี่ยนได้ขณะกำลังบันทึกเสียง |
| 🏳️ ธงชาติฝั่งขวา | ธงชาติของภาษาอีกฝ่าย |
| 🔴 **แถบสีแดง LIVE** | การเชื่อมต่อเสียงเรียลไทม์ปกติ แปลผ่าน Gemini Live API |
| ⚡ **แถบสีส้ม Local** | โหมดสำรอง ใช้ระบบจำเสียงในเครื่อง การแปลยังทำงานได้ปกติ |
| 🎙️ ปุ่มบันทึกเสียง | 🔵 สีน้ำเงิน = ยังไม่ได้บันทึก คลิกเพื่อเริ่ม; 🔴 กรอบแดงกะพริบ = กำลังบันทึก คลิกเพื่อหยุด |

> [!IMPORTANT]
> ต้องเลือกภาษาให้เรียบร้อย**ก่อนเริ่มบันทึกเสียง** เนื่องจากไม่สามารถเปลี่ยนขณะใช้งานได้

### 3. กล่องข้อความการแปล

- หน้าจอจะแสดงชื่อผู้พูด (เช่น `Barret`), ข้อความต้นฉบับ และข้อความที่แปลแล้ว
- แถบสถานะด้านบน:
  - 🔴 **"กำลังฟัง..."** = Gemini Live กำลังประมวลผลเสียงเรียลไทม์
  - 🟡 **"โหมด Local กำลังฟัง..."** = อยู่ในโหมดสำรอง ระบบยังสามารถจำเสียงและแปลได้ตามปกติ
- ปุ่มด้านขวาบน:
  - **แชร์ (Share)**: คัดลอกหรือแชร์บันทึกการสนทนาทั้งหมด
  - **ล้างข้อมูล (Clear)**: ลบประวัติการสนทนาทั้งหมด (เฉพาะผู้สร้างห้องเท่านั้นที่ทำได้)

---

### 4. ช่องพิมพ์ข้อความ (ด้านล่างสุด)

กรณีที่ไม่สะดวกใช้เสียง หรือต้องการป้อนข้อความให้แม่นยำ:

1. คลิกปุ่มทิศทางด้านซ้ายเพื่อเลือกทิศทางการแปล (Local → Client หรือ Client → Local)
2. พิมพ์ข้อความในช่อง
3. กด Enter หรือคลิกปุ่มส่งเพื่อแปลภาษา

> [!NOTE]
> คุณสามารถใช้งานการพิมพ์ข้อความได้ตลอดเวลา **โดยไม่จำเป็นต้องเปิดโหมดบันทึกเสียง**

---

### 5. ตั้งค่าเสียงพูด (ปุ่มลอยขวาล่าง)

คลิกปุ่มวงกลม (รูปลำโพง) เพื่อเปิดการตั้งค่าเสียงอ่าน:

| ตัวเลือก | คำอธิบาย |
|---|---|
| **ปิดเสียง (None)** | ไม่เล่นเสียงอ่านของผลการแปล |
| **เฉพาะตัวเอง (Myself)** | เล่นเสียงอ่านเฉพาะในเครื่องของคุณเท่านั้น |
| **ทั้งหมด (ALL)** | ผู้เข้าร่วมทุกคนจะได้ยินเสียงอ่าน |
| **เฉพาะคนอื่น (Others)** | เฉพาะผู้เข้าร่วมคนอื่นจะได้ยินเสียงอ่าน ตัวคุณจะไม่ได้ยิน |

---

### 6. แถบเครื่องมือด้านบน

ด้านซ้ายบนแสดงตราสัญลักษณ์ **TUC** (ตัวอักษรสีแดงตัวหนา) และชื่อรอง (ค่าเริ่มต้น: Equipment Department) ด้านขวาเรียงตามลำดับ:

| ไอคอน | ฟังก์ชัน |
|---|---|
| 📋 Room ID + ปุ่มคัดลอก | คัดลอกลิงก์เชิญเพื่อส่งให้ผู้อื่นเข้าร่วมห้อง |
| 📱 ไอคอน QR Code | แสดง QR Code สำหรับให้ผู้อื่นสแกนเข้าร่วม |
| 🚪 ไอคอนออกจากระบบ | ออกจากห้องปัจจุบัน (ผู้สร้างห้องสามารถเลือก "ปิดห้องประชุม" ได้) |
| 👥 ไอคอนจำนวนคน (xx/100) | จำนวนผู้ใช้งานในระบบขณะนี้ |
| 🌙 / ☀️ ไอคอนพระจันทร์/พระอาทิตย์ | สลับโหมดมืด (Dark Mode) / โหมดสว่าง (Light Mode) |
| 🔒 ไอคอนแม่กุญแจ | การตั้งค่าผู้ดูแล (แสดงเฉพาะผู้สร้างห้อง) |
| 🟢 **ระบบพร้อม** | สถานะระบบทำงานปกติ |

---

### 7. การตั้งค่าผู้ดูแล (ไอคอนแม่กุญแจ)

เข้าถึงได้เฉพาะผู้สร้างห้อง ประกอบด้วย:

- **Quotas Limitation**: กราฟแสดงการใช้งาน RPM / RPD ของ API Key ปัจจุบัน
- **ตั้งค่า API Key**: เปลี่ยนหรืออัปเดต Gemini API Key หรือ Google Cloud API Key
- **สลับแพ็กเกจ API**: เลือกโหมด Free Tier (ฟรี) หรือ Tier 1 (จ่ายตามจริง) เพื่อให้กราฟแสดงผลถูกต้อง
- **ตั้งค่าหัวเรื่อง**: ปรับแต่งชื่อแบรนด์ซ้าย (ค่าเริ่มต้น: `TUC`) และชื่อรอง (ค่าเริ่มต้น: `Equipment Department`)

---

### 8. ทำไมต้องใช้ API Key ส่วนตัว? (BYOK)

ระบบนี้ใช้โมเดล **BYOK (Bring Your Own Key)** ซึ่งมีข้อดีที่สำคัญดังนี้:

1. **โควตาเป็นอิสระต่อกัน**: รหัสฟรีแต่ละรหัสมีขีดจำกัดการเชื่อมต่อ 2 ครั้งต่อนาที (RPM) หากคนจำนวนมากใช้รหัสเดียวกันร่วมกัน จะทำให้ถึงขีดจำกัดได้ง่ายและส่งผลให้ทุกคนหลุดจากการเชื่อมต่อ การใช้รหัสของคุณเองจะช่วยให้การเชื่อมต่อเสถียรและไม่ถูกรบกวนจากผู้อื่น
2. **สมัครใช้งานได้ง่ายและฟรี**: คุณสามารถขอรหัสฟรีได้ภายในเวลาไม่ถึงนาทีผ่าน [Google AI Studio](https://aistudio.google.com/app/apikey) โดยไม่ต้องใช้บัตรเครดิตหรือตั้งค่าที่ซับซ้อน
3. **ความเสถียรของการเชื่อมต่อ**: การมีโควตาส่วนตัวช่วยให้มั่นใจได้ว่าคุณจะสามารถใช้งานการจำเสียงและการแปลที่เสถียรในช่วงเวลาที่มีการใช้งานสูงหรือในการประชุมที่ยาวนาน
4. **ความเป็นส่วนตัวของข้อมูล**: รหัส API และประวัติการสนทนาของคุณจะถูกเก็บไว้ในเบราว์เซอร์ของคุณเท่านั้น รหัสจะสื่อสารโดยตรงกับเซิร์ฟเวอร์ของ Google โดยไม่ผ่านเซิร์ฟเวอร์ตัวกลางของบุคคลที่สาม

## 2. ข้อจำกัดการใช้งาน

### 1. ข้อจำกัดโควตา API

ระบบนี้ใช้ Google Gemini API ข้อจำกัดสำหรับบัญชี Free Tier (ฟรี) มีดังนี้:

| รายการ | ข้อจำกัด Free Tier |
|---|---|
| **จำนวนการเชื่อมต่อต่อนาที (RPM)** | 2 ครั้ง |
| **จำนวนการเชื่อมต่อต่อวัน (RPD)** | 50 ครั้ง |
| **ระยะเวลาเชื่อมต่อต่อเนื่องสูงสุด** | 3 ชั่วโมง (ระบบจะตัดการเชื่อมต่ออัตโนมัติ) |
| **การตัดการเชื่อมต่อเมื่อไม่มีการใช้งาน** | แจ้งเตือนหลัง 1 ชม. และตัดการเชื่อมต่อหากไม่ตอบสนองใน 3 นาที |

เวลาล้างโควตา (Reset):
- RPM: คำนวณแบบย้อนหลังทุก 1 นาที (ไม่ใช่เวลาคงที่)
- RPD: ล้างโควตาทุกวันเวลา 15:00 น. (ฤดูหนาว) หรือ 16:00 น. (ฤดูร้อน) ตามเวลาประเทศไทย

### 3. การรองรับภาษา

ระบบรองรับ 37 ภาษา อย่างไรก็ตาม ใน**โหมดสำรอง (Local STT)** ความแม่นยำอาจลดลงในบางภาษา แนะนำให้ใช้ Chrome หรือ Safari หากสื่อสารด้วยภาษาเหล่านี้:
- ภาษาอาหรับ (Arabic)
- ภาษาฮีบรู (Hebrew)
- ภาษาฟิลิปปินส์ (Filipino)

## 3. สถานการณ์การใช้งานปกติและเทคนิคการเลี่ยงโควตา

### ข้อมูลพื้นฐาน

จำนวน "การเชื่อมต่อ" ของ Gemini Live จะถูกนับเมื่อ: **เริ่มบันทึกเสียงแต่ละครั้ง หรือเมื่อระบบเชื่อมต่อใหม่อัตโนมัติ** (ไม่เกี่ยวกับระยะเวลาที่พูด)

- **การประชุม 1 ครั้งต่อเนื่อง** (เริ่มบันทึกครั้งเดียว ไม่หลุดระหว่างทาง) = ใช้ **1 RPD**
- **การเชื่อมต่อใหม่อัตโนมัติ 1 ครั้ง** = ใช้เพิ่มอีก **1 RPD และ 1 RPM**

---

### ตัวอย่างสถานการณ์การใช้งานปกติ

| พฤติกรรม | จำนวนที่ใช้ |
|---|---|
| เริ่มบันทึก 1 ครั้ง ใช้งานต่อเนื่อง 3 ชม. | 1 RPD |
| ครบ 3 ชม. ระบบตัดแล้วเริ่มใหม่เอง | เพิ่มอีก 1 RPD |
| สลับภาษาแล้วเริ่มใหม่ | +1 RPD |
| อินเทอร์เน็ตหลุด ระบบเชื่อมต่อใหม่สำเร็จ | +1 RPD (อัตโนมัติ) |

**การประเมินการใช้งาน 50 RPD ต่อวัน:**

หากการประชุมแต่ละครั้งมีการเชื่อมต่อใหม่ 2-3 ครั้ง คุณจะสามารถใช้งานได้วันละประมาณ **16-25 การประชุม**

---

### คำแนะนำเพื่อไม่ให้ติดข้อจำกัด (Quota)

> [!IMPORTANT]
> **ตั้งค่าภาษาให้เรียบร้อยก่อนเริ่ม**: การเปลี่ยนภาษาจะทำให้ระบบต้องเชื่อมต่อใหม่ ทุกครั้งที่เปลี่ยนจะเสีย 1 RPD (ระบบจะล็อคการสลับขณะบันทึก)

> [!IMPORTANT]
> **เลี่ยงการกดเริ่ม/หยุดบันทึกซ้ำๆ ใน 1 นาที**: บัญชีฟรีอนุญาตให้เชื่อมต่อได้เพียง 2 ครั้งต่อนาที หากกดเล่น/หยุดเกิน 3 ครั้ง ระบบจะขึ้นเตือน "API ไม่พร้อมใช้งานชั่วคราว" ต้องรอ 1 นาทีจึงจะใช้งานได้ใหม่

1. **ใช้อินเทอร์เน็ตที่เสถียร**: สัญญาณที่ขาดหายจะทำให้ระบบพยายามเชื่อมต่อใหม่บ่อยครั้ง การใช้งานผ่าน Wi-Fi จะเสถียรกว่าเน็ตมือถือ

2. **แนะนำให้บันทึกต่อเนื่องสำหรับการประชุมยาว**: อย่ากดหยุดเพียงเพราะหยุดพูดชั่วคราว เพื่อเลี่ยงการเสียโควตาในการเริ่มเซสชันใหม่

3. **เมื่อขึ้น ⚡ Local (แถบสีส้ม) ถือเป็นปกติ**: หมายถึงระบบสลับไปใช้ตัวจำเสียงในเครื่องแทน **ฟังก์ชันการแปลยังทำงานได้** ระบบจะพยายามเชื่อมต่อคลาวด์ใหม่ที่เบื้องหลัง และจะสลับกลับเป็น 🔴 LIVE โหมดปกติเองเมื่อสำเร็จ
