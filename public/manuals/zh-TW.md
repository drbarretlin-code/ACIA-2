# ACIA-2 AI 即時翻譯系統 使用說明書

---

## 一、介面功能與操作說明

### 1. 進入系統

首次開啟網頁後，依序完成以下步驟：

1. **輸入名稱**：填入您的顯示名稱（例如：`Barret`、您的職銜或座位號碼）。
2. **輸入 Gemini API Key**：填入您的 Google AI Studio API 金鑰（格式為 `AIza...`）。
3. **建立或加入會議室**：
   - 點擊「建立新會議室」開啟翻譯工作階段。
   - 或在網址列貼上他人分享的邀請連結直接加入。

> [!NOTE]
> API Key 儲存於瀏覽器本機，不會上傳至任何伺服器。

---

### 2. 語言選擇列（控制面板）

畫面中央橫列，由左至右依序為：

| 元素 | 說明 |
|---|---|
| 🏳️ 左側國旗 | 您自己說話的語言（來源語言）|
| 🌐 左側下拉選單 | 選擇「我的語言」（Local），錄音期間無法變更 |
| ⇄ 中間箭頭按鈕 | 快速互換左右語言，錄音期間無法操作 |
| 🌐 右側下拉選單 | 選擇「對方語言」（Client），錄音期間無法變更 |
| 🏳️ 右側國旗 | 對方的語言旗幟 |
| 🔴 **LIVE 紅色標籤** | 即時語音連線正常，透過 Gemini Live API 翻譯 |
| ⚡ **Local 琥珀色標籤** | 備援模式，使用本地語音辨識，翻譯仍正常運作 |
| 🎙️ 錄音按鈕 | 🔵 藍色 = 未錄音，點擊開始；🔴 閃爍紅框 = 錄音中，點擊停止 |

> [!IMPORTANT]
> 語言必須在**開始錄音之前**設定完成，錄音中無法切換。

---

### 3. 翻譯對話框

- 每一則對話會顯示：發話人名稱（例如：`Barret`）、原文、譯文。
- 對話框標題列狀態：
  - 🔴 **正在聆聽...** = Gemini Live 即時語音處理中
  - 🟡 **本地模式聆聽中...** = 備援模式中，仍可正常辨識翻譯
- 右上角按鈕：
  - **分享**：將目前所有對話紀錄複製或分享
  - **清除**：刪除所有對話紀錄（房間建立者才能操作）

---

### 4. 文字輸入列（畫面底部）

當無法使用語音，或需要精確輸入文字時：

1. 點擊左側方向按鈕切換翻譯方向（Local → Client 或 Client → Local）。
2. 在文字框輸入內容。
3. 按 Enter 或點擊送出按鈕翻譯。

> [!NOTE]
> 文字輸入任何時候都可使用，**不需要先開啟錄音功能**。

---

### 5. 語音輸出設定（右下角懸浮按鈕）

點擊右下角圓形按鈕（喇叭圖示）開啟播放設定：

| 選項 | 說明 |
|---|---|
| **靜音** | 翻譯結果不播放語音 |
| **僅自己** | 只在自己的裝置播放譯文語音 |
| **全部** | 所有參與者都能聽到語音播放 |
| **僅他人** | 只有其他參與者聽到，自己不播放 |

---

### 6. 頂部工具列

頂部左側顯示品牌標識 **TUC**（紅色粗體）與副標題（預設：Equipment Department）。右側工具列依序：

| 圖示 | 功能 |
|---|---|
| 📋 房間 ID + 複製按鈕 | 複製邀請連結，分享給其他人加入 |
| 📱 QR Code 圖示 | 顯示 QR Code，讓他人掃描加入 |
| 🚪 登出圖示 | 離開目前房間（房間建立者可選擇「結束會議室」）|
| 👥 人數圖示（xx/100）| 目前系統在線人數 |
| 🌙 / ☀️ 月亮 / 太陽圖示 | 切換深色 / 淺色模式 |
| 🔒 鎖頭圖示 | 管理者設定（僅房間建立者可見）|
| 🟢 **系統就緒** | 系統正常運作中 |

---

### 7. 管理者設定（鎖頭圖示）

僅房間建立者可進入，包含：

- **Quotas Limitation**：查看目前 API Key 的 RPM / RPD 使用量儀表板。
- **API Key 設定**：更換 Gemini API Key 或 Google Cloud API Key。
- **API 方案切換**：Free Tier（免費）或 Tier 1（付費）配額對應。
- **頂部標題設定**：自訂左側品牌名（預設：`TUC`）與副標題（預設：`Equipment Department`）。

## 二、使用限制說明

### 1. API 配額限制

本系統使用 Google Gemini API，Free Tier（免費）帳號的限制如下：

| 項目 | Free Tier 限制 |
|---|---|
| **每分鐘連線次數（RPM）** | 2 次 |
| **每日連線次數（RPD）** | 50 次 |
| **單次連線最長時間** | 3 小時（系統強制斷線）|
| **閒置自動斷線** | 1 小時後詢問是否繼續，3 分鐘無回應自動斷線 |

配額重置時間：
- RPM：每分鐘滾動重置（非固定整點）
- RPD：每日台灣時間下午 15:00（冬令）或 16:00（夏令）重置

---

### 3. 語言支援說明

系統支援 37 種語言。部分語言在**備援模式（本地 STT）**下的辨識準確度較低，建議優先使用 Chrome 或 Safari：
- 阿拉伯語（Arabic）
- 希伯來語（Hebrew）
- 菲律賓語（Filipino）

---

### 4. iOS 裝置特別說明

> [!WARNING]
> **切換 App 至背景或鎖定螢幕**：麥克風會被 iOS 系統中斷。系統會在您回到 App 後 2 秒內自動重新啟動錄音，若未自動恢復，請手動重新點擊錄音按鈕。

- **接聽電話中**：錄音功能會暫停，掛斷後請手動重新啟動。

## 三、正常使用情境與避免觸發配額限制

### 背景說明

Gemini Live 即時語音的「連線次數」計算方式是：**每次啟動錄音、或系統自動重連，即算 1 次請求**。與您說了多少話無關。

- **一場完整會議**（啟動一次錄音，中途未斷線）= 消耗 **1 次 RPD**
- **自動重連一次** = 額外消耗 **1 次 RPD 和 RPM**

---

### 正常使用下的典型場景

| 行為 | 消耗次數 |
|---|---|
| 開始錄音 1 次，順利使用 3 小時 | 1 RPD |
| 3 小時到期，系統斷線後手動重新開始 | 再 +1 RPD |
| 語言切換後重新開始 | +1 RPD |
| 網路瞬斷，系統自動重連成功 | +1 RPD（自動）|

**每日 50 次 RPD 的實際估算：**

若每次會議平均消耗 2-3 次（含重連），每天可進行約 **16-25 場會議**。

---

### 避免觸及限制的操作建議

> [!IMPORTANT]
> **語言在開始錄音前設定好**：語言切換會觸發重連，每切換一次 +1 RPD。錄音期間語言選單已鎖定，必須先停止錄音再變更。

> [!IMPORTANT]
> **避免在 1 分鐘內啟停錄音超過 2 次**：Free Tier 每分鐘只允許 2 次連線。若反覆按下啟動 / 停止，第 3 次會被 API 拒絕並顯示「API 暫時不可用」提示，需等待下一分鐘才能重試。

1. **保持穩定的網路環境**：網路瞬斷會觸發自動重連機制。Wi-Fi 環境比行動網路更穩定。

2. **長時間會議建議連續錄音**：啟動一次錄音持續使用，勿因短暫停頓就按停止，避免稍後再啟動時多消耗配額。

3. **出現 ⚡ Local（琥珀色）標籤是正常的**：這代表系統已切換至本地辨識，翻譯功能**仍然正常運作**，系統會在後台自動嘗試重新連線，成功後自動切回 🔴 LIVE 雲端模式。

---

### 狀態提示訊息說明

| 提示訊息 | 意義 | 建議行動 |
|---|---|---|
| 「翻譯工作階段已結束，系統正在自動重新建立連線。」| Session 正常到期（常見於長時間使用後）| 等候系統自動重連即可 |
| 「API 暫時不可用（可能已達使用上限），已切換至本地模式。」| 觸及 RPD 或 RPM 限制 | 等待下一分鐘或明日重置後繼續 |
| 「連線暫時中斷，已切換至本地模式。語音與文字輸入仍可正常使用。」| 網路不穩定導致的斷線 | 等候系統自動恢復，期間可繼續說話或文字輸入 |
| 「麥克風已中斷，請重新啟動錄音。」| iOS 系統強制中斷麥克風 | 點擊錄音按鈕重新啟動 |
